# Animation plans

Generated by an `improve-animations`-style audit on commit `17127a5`. Covers
the 3 MEDIUM-severity findings selected from the full audit; the 2 LOW
findings (dialog/select entrance duration, checkbox check-in transition) were
not selected and have no plan file.

| # | Title | Severity | Status |
| --- | --- | --- | --- |
| [001](001-animate-category-collapse.md) | Animate category collapse/expand instead of teleporting | MEDIUM | DONE |
| [002](002-replace-transition-all.md) | Replace `transition-all` with explicit properties on Button/Badge/Tabs | MEDIUM | DONE |
| [003](003-add-reduced-motion-support.md) | Add global `prefers-reduced-motion` support | MEDIUM | DONE |

## Recommended execution order

1. **002** first — zero behavioral risk (pure property-list narrowing, no
   visual change expected), fastest to verify, and touches the components
   003's feel-check will exercise (buttons, tabs).
2. **001** second — the highest-impact fix, but it changes the DOM structure
   of the category panel (always-rendered instead of conditionally mounted),
   so it's worth landing and feel-checking on its own before layering the
   reduced-motion rule on top.
3. **003** last — the reduced-motion feel-check in this plan explicitly
   re-verifies both the dialog (unaffected by 001/002) and the category
   collapse (from 001), so running it last means both prior fixes exist to
   check against.

No file-level dependencies between the three (each touches a disjoint set of
files: `dashboard-client.tsx` / `button.tsx`+`badge.tsx`+`tabs.tsx` /
`globals.css`), so they can also be done in any order or in parallel by
different executors — the order above is about verification convenience,
not a hard requirement.
